### Why this happens
When you run `pip install`, it sometimes installs to a user-specific folder or a different Python environment than the one triggered by `python`. Using `python -m pip` ensures you are installing packages specifically for the Python interpreter that will execute your script. 

If you are seeing this error locally, simply run `pip install -r requirements.txt` in your terminal to fix it on your machine.
